## 수정 제안 목록

### [P1] C_04_018 -- 난이도 과도하게 낮음 + 오답 매력도 부족

- **대상**: C_04_018 전체
- **우선순위**: P1
- **현재 내용**: DB Link를 통한 원격 테이블 참조 구문 (table@dblink_name)
- **문제점**: 
  - 선택지 B(#), C(dblink.table), D(USING)가 SQL을 조금이라도 아는 수험자에게 즉시 제거 가능한 명백한 오답
  - "@" 기호를 사용한다는 것은 Oracle DBA 시험의 가장 기초적인 지식으로, 원본 시험 및 기존 커스텀 문제의 난이도에 비해 현저히 쉬움
  - Group A 챕터에 부적절한 난이도
- **수정 제안**: DB Link의 활용을 더 깊이 있게 묻는 문제로 교체. 예시: (1) DB Link를 통한 원격 테이블 참조 시 optimizer의 동작 (DRIVING_SITE 힌트), (2) DB Link 접속 실패 시 오류 진단 절차, (3) SHARED DB Link와 일반 DB Link의 차이.

### [P1] C_04_017 -- C_04_006과 동일 개념 중복

- **대상**: C_04_017 전체
- **우선순위**: P1
- **현재 내용**: 리스너 미실행 시 ORA-12541 발생
- **문제점**: C_04_006에서 이미 tnsping으로 TNS-12541(no listener)을 해석하는 문제를 출제함. ORA-12541과 TNS-12541은 동일한 오류이며, "리스너가 미실행 → no listener 오류 발생"이라는 동일한 학습 포인트. 형식만 tnsping에서 sqlplus로 바뀌었을 뿐.
- **수정 제안**: 리스너 관련이지만 다른 관점의 문제로 교체. 예시: (1) lsnrctl services 출력에서 핸들러 정보 해석, (2) REMOTE_LISTENER 파라미터의 RAC 환경에서의 역할, (3) 리스너 로깅 설정 및 listener.log 분석.

### [P2] C_04_019 -- C_04_014와의 주제 중복

- **대상**: C_04_019 전체
- **우선순위**: P2
- **현재 내용**: CREATE DATABASE LINK 구문에서 Private/Public 구분, USING 절, CONNECT TO 생략 시 동작
- **문제점**: C_04_003에서 Private vs Public 구분을, C_04_014에서 Fixed User / Connected User / Current User 접속 방식을 이미 상세히 출제. C_04_019는 이 두 문제의 지식을 조합한 것이지만 새로운 학습 포인트가 적음. 단, ALTER DATABASE LINK로 비밀번호 변경 불가라는 선택지 C의 내용은 C_04_014에서도 다루고 있어 중복.

### [P3] C_04_016 -- 선택지 A의 오답 해설 정확성 보완

- **대상**: C_04_016 선택지 A 해설
- **우선순위**: P3
- **현재 내용**: "상태가 READY로 표시되는 것은 동적 서비스 등록의 특징입니다. 정적 서비스 등록의 경우 리스너가 인스턴스의 실제 상태를 알 수 없으므로 상태가 UNKNOWN으로 표시됩니다."
- **문제점**: 해설 자체는 정확하나, 선택지 A에서 "pdb1는 정적 서비스 등록"이라고 주장하고 있으므로, 해설에서 "pdb1의 상태가 READY이므로 동적 등록이며, 정적 등록이라면 UNKNOWN으로 표시됨"이라고 보다 직접적으로 반박하면 이해가 쉬울 것.
- **수정 제안**: 해설을 "pdb1 서비스의 상태가 READY로 표시되어 있으므로 이는 동적 서비스 등록에 의한 것입니다. 정적 서비스 등록이었다면 상태가 UNKNOWN으로 표시됩니다. 따라서 pdb1이 정적 등록이라는 설명은 틀립니다."로 보완.
