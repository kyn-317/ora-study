## 수정 제안 목록

### [P3] C_01_020 -- Redo Log Buffer 문제가 기존 문제와 유사한 개념 영역

- **대상**: C_01_020 전체
- **우선순위**: P3
- **현재 내용**: Redo Log Buffer의 순환 버퍼 구조, LOG_BUFFER 파라미터, COMMIT 여부 무관 기록, LGWR 역할
- **문제점**: C_01_007(LGWR 기록 조건)에서 이미 LGWR의 역할, COMMIT 시 기록, Write-Ahead Logging 등을 다루고 있어 학습 포인트가 상당히 겹침. 또한 원본 문제 01_005 선택지 A("데이터베이스 ・バッファ・キャッシュは、循環バッファとして使用され")에서 순환 버퍼 개념이 언급됨.
- **수정 제안**: 기존 출제되지 않은 주제(예: SGA Granule 개념, PGA의 SQL 작업 영역, Streams Pool)로 교체 검토. 현재 문제 자체의 품질은 우수하므로 유지해도 무방하나, 주제 다양성 관점에서 교체 권장.

### [P2] C_01_019 -- FORCE 옵션 정확성 보완

- **대상**: C_01_019 선택지 A 및 해설
- **우선순위**: P2
- **현재 내용**: "ALTER PLUGGABLE DATABASE pdb_name OPEN READ WRITE FORCE;"
- **문제점**: FORCE 옵션은 정확한 동작이나, 선택지 B의 "ALTER DATABASE pdb_name OPEN READ WRITE"도 일부 Oracle 버전에서는 CDB$ROOT에서 실행 시 유사하게 동작할 수 있어 혼동 가능성이 있음. 해설에서 "ALTER DATABASE는 CDB 전체를 대상으로 하는 명령"이라는 설명은 단순화된 설명이며, ALTER DATABASE ... OPEN 구문은 CDB$ROOT에서 PDB를 열 때도 사용될 수 있음.
- **수정 제안**: 선택지 B의 오답 해설을 "ALTER DATABASE는 PDB의 오픈 모드 변경에 사용하는 표준 구문이 아닙니다. PDB 오픈 모드 변경은 ALTER PLUGGABLE DATABASE를 사용하는 것이 올바른 방법입니다."로 수정하여 보다 정확한 설명 제공.
